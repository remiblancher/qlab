WHY DIFFERENT ALGORITHMS AT EACH LEVEL?
---------------------------------------

┌─────────────┬──────────────────────────────────────────────────────────────────┐
│             │                                                                  │
│  ROOT CA    │  • Longest-lived (10-25 years)                                   │
│  ML-DSA-87  │  • Highest security requirement                                  │
│  Level 5    │  • Larger size acceptable (rarely transmitted)                   │
│             │  • "Crown jewels" of the PKI                                     │
│             │                                                                  │
├─────────────┼──────────────────────────────────────────────────────────────────┤
│             │                                                                  │
│ ISSUING CA  │  • Medium lifespan (3-10 years)                                  │
│  ML-DSA-65  │  • Balance security and performance                              │
│  Level 3    │  • Good signature size for embedding in certificates             │
│             │  • ~192-bit security is plenty for most threats                  │
│             │                                                                  │
├─────────────┼──────────────────────────────────────────────────────────────────┤
│             │                                                                  │
│ END-ENTITY  │  • Short lifespan (90 days - 2 years)                            │
│  ML-DSA-65  │  • Transmitted frequently (every TLS connection)                 │
│  + ML-KEM   │  • ML-KEM for key exchange (TLS 1.3 handshake)                   │
│  Level 3    │  • Dual algorithm: signing + encryption                          │
│             │                                                                  │
└─────────────┴──────────────────────────────────────────────────────────────────┘
